# Abstract

In this talk author will share his personal experience how to use GNU make in CI/CD workflow and how this approach might be beneficial compared to use of external CI/CD tools in some cases.

Author will provide a lot of practical examples and provide comparison on the tools with

Overview:

The debate between traditional build tools like GNU Make and modern CI/CD pipelines continues to intrigue developers and engineers alike. While CI/CD tools offer automation and integration advantages, GNU Make remains a stalwart for its simplicity and flexibility in managing build processes. This session aims to explore the strengths of GNU Make in comparison to CI/CD tools, highlighting scenarios where Make emerges as the preferred choice.

Session Focus:

This session seeks proposals that delve into the practical applications, benefits, and best practices of using GNU Make over CI/CD tools such as Jenkins, GitLab CI, GitHub Actions.

Topics of interest include but are not limited to:

Case studies demonstrating the efficiency of GNU Make in complex build environments.

Integrating GNU Make with CI/CD pipelines for optimal build management.

Scalability and performance considerations when using GNU Make at scale.

Real-world examples showcasing scenarios where GNU Make outperforms CI/CD tools.

This talk is aimed at developers, engineers, DevOps practitioners, and anyone with experience or insights into the strengths of GNU Make in build automation and management.
